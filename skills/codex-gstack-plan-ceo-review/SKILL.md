# codex-gstack-plan-ceo-review

Use this skill to challenge scope and product framing before code starts.

## Command

```bash
$HOME/.codex/gstack-macos/bin/gstack-workflow-plan-ceo-review --repo /path/to/target-repo [--initiative-id <initiative-id>]
```

## Review Focus

- Is the problem framed correctly?
- What is the smallest wedge worth shipping first?
- Which goals should be removed from v1?
- What decision would a founder make to keep momentum high?

## Output

- Update the current initiative plan under `docs/gstack/<initiative-id>/plan.md`
- Leave a concise CEO Review section that sharpens scope instead of expanding it by default
